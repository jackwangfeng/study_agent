# 高中学习监督Agent - PRD文档

## 1. 产品概述

### 1.1 产品定位

**产品名称**：智学伴 (StudyPal)

**一句话说清楚**：一个懂高中生心理、专治"努力了没进步"的AI错题陪练

### 1.2 核心价值

| 维度 | 描述 |
|------|------|
| **目标用户** | 高中阶段想努力但方法不对、成绩中等的学生 |
| **解决什么问题** | 错过的题反复错、努力了看不到进步、学习孤独感 |
| **核心差异** | 持续追踪错题直到真正掌握，而非一次性答疑 |

### 1.3 产品口号

> "陪你搞定每一道错题，见证每一次进步"

---

## 2. 市场背景

### 2.1 目标用户画像

**典型用户A：小美，高二女生**
- 成绩：中游偏上（年级200名/800人）
- 痛点：认真做了错题本但从不回头看，下次遇到还是错
- 期望：希望有人帮她追踪错题，而不是抄一遍就完事

**典型用户B：小强，高三男生**
- 成绩：中等（年级400名）
- 痛点：考砸了不敢跟爸妈说，自己扛着，越扛越焦虑
- 期望：希望有人理解他的压力，给他鼓励

### 2.2 竞争分析

| 竞品 | 核心能力 | 不足 |
|------|----------|------|
| 豆包AI | 答疑、讲题、作文批改 | 无记忆、一次性对话、不追踪错题 |
| 小猿搜题 | 拍照搜题、视频讲解 | 无AI互动、无情绪支持 |
| 错题打印机 | 打印错题 | 无智能归类、无追踪 |

**我们的机会**：做"有记忆的AI错题本"，持续追踪+情绪陪伴

## 4. 用户旅程

### 4.1 主路径：错题学习

```
用户打开微信
    ↓
发送"拍题"或直接发送错题照片
    ↓
AI识别题目，显示"已收到题目"
    ↓
询问用户："这道题你是哪里卡住了？"
    ↓
用户描述困惑点
    ↓
AI进行引导式讲解
    ↓
讲解完成后询问："下次遇到同类题能做对吗？"
    ↓
记录错题到个人错题本
    ↓
若某知识点错题≥3次，提示"这个知识点要加强哦"
```

### 4.2 每日学习规划路径

```
用户发送"今日计划"或"开始学习"
    ↓
AI根据薄弱点和期末目标生成今日学习计划
    ↓
显示计划列表："1. 复习函数图像 2. 搞定3道三角函数错题..."
    ↓
用户确认或调整计划
    ↓
进入番茄钟专注模式（25分钟学习+5分钟休息）
    ↓
每个番茄钟结束记录进度
    ↓
全天学习结束，生成学习日报
    ↓
询问用户："明天想重点突破哪科？"
```

### 4.3 情绪支持路径

```
用户发送"今天考砸了" / "学不进去" / "不想学了"
    ↓
AI识别情绪关键词
    ↓
进行情绪支持对话
    ↓
若判断需要更多帮助，引导至学习辅导
```

### 4.4 家长端路径

```
家长关注公众号并绑定孩子账号
    ↓
每周六收到孩子学习周报推送
    ↓
周报内容：本周学习时长、完成任务、薄弱点、情绪状态
    ↓
如发现异常（如连续3天未学习），自动推送提醒
    ↓
家长可回复"查看详情"获取更多信息
    ↓
家长可设定学习目标，孩子端会收到
```

## 6. AI能力要求

### 6.1 错题识别

- 准确识别印刷体题目（准确率≥90%）
- 手写体题目提供模糊匹配建议
- 识别失败时提示用户重新拍照

### 6.2 引导式讲解

- 不直接给答案
- 通过提问引导思考
- 讲解分步骤，给学生消化时间
- 讲解后要求学生复述理解

### 6.3 情绪识别

关键词识别：
- 负面：考砸/退步/放弃/焦虑/累/烦
- 正面：进步/开心/有动力

情绪回复原则：
- 先共情，再分析
- 不批评，只鼓励
- 不讲大道理，用朋友的口吻

## 8. 风险与挑战

| 风险 | 应对策略 |
|------|----------|
| AI识别率低 | 提供手动输入题目选项 |
| 用户新鲜感消退 | 设置成就系统、学习天数连续打卡 |
| 免费用户转化低 | 提供免费体验次数，控制高级功能 |
| 情感识别不准确 | 优先文本关键词识别，图片识别后期再做 |

## 10. 附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| 引导式讲解 | 不直接给答案，通过提问引导学生自主思考解题 |
| 薄弱点 | 同一个知识点错误次数≥3次 |
| 遗忘曲线 | 艾宾浩斯遗忘曲线，错题需在1/3/7天后复习 |
| 会员权益 | 不同订阅等级可使用的功能差异 |

### 10.2 技术备注

- 图片存储：七牛云/腾讯云COS
- OCR识别：Tesseract.js（本地）或腾讯云OCR
- AI对话：OpenAI GPT-3.5/GPT-4
- 数据库：MongoDB
- 缓存：Redis
